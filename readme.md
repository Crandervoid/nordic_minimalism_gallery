## üí¨ How Vibe Coding Works

**Traditional:** You write code ‚Üí Test ‚Üí Debug ‚Üí Repeat  
**Vibe Coding:** You describe ‚Üí AI codes ‚Üí You screenshot ‚Üí Review ‚Üí Refine

### Key: AI Is Blind üëÅÔ∏è

AI can't see what it built. You provide screenshots constantly.

**Your job:** Design judgment, screenshots, decisions  
**AI's job:** Code execution, technical quality

## ‚úÖ Success Checklist

**Do:**
- Take screenshots constantly (desktop, tablet, mobile)
- Iterate 2-3x per section before moving on
- Compare your work to reference images
- Stop AI when it goes off track
- Track your time (portfolio gold!)

**Don't:**
- Accept AI's first output
- Skip mobile testing
- Forget screenshots for reviews
- Write code manually
- Play it safe with design
